### 游戏逻辑修复 (fix)

#### 弃牌玩家跳过
- **问题描述**
  - 弃牌或全下的玩家仍被轮到行动
  - 导致游戏流程卡顿

- **修复方案**
  ```javascript
  // backend/games/poker/poker.socket.js
  function notifyCurrentPlayer(game, io) {
    const player = game.players[game.currentPlayer];
    if (!player) return;
    
    // 跳过弃牌或全下的玩家
    if (player.folded || player.allIn) {
      const nextPlayer = game.findNextActivePlayer(game.currentPlayer);
      if (nextPlayer >= 0) {
        game.currentPlayer = nextPlayer;
        notifyCurrentPlayer(game, io);
      }
      return;
    }
    // ...后续逻辑
  }
  ```

#### 手牌评估Bug修复
- **问题描述**
  - 牌型判断函数中 cards[0] 可能为 undefined
  - 导致服务器报错

- **修复方案**
  ```javascript
  // backend/games/poker/PokerGame.js
  function isFlush(cards) {
    if (!cards || cards.length < 5) return false;
    const suit = cards[0].suit;
    return cards.every(c => c.suit === suit);
  }
  // 其他牌型函数类似添加长度检查
  ```

## [v1.1.0] - 2026-03-22

### 人机对战系统 (feat)

#### 新增功能
- **人机创建机制**
  - 房主可在等待房间添加人机玩家
  - 支持最多20个不同头像的人机
  - 人机自动准备，无需等待
  - 人机头像路径: `frontend/avatars/bots/bot_1.png` ~ `bot_20.png`

- **人机AI决策系统**
  - 自动决策逻辑: fold/call/check/raise/allin
  - 模拟思考时间: 1-3秒随机延迟
  - 策略: 手牌好则加注，一般则跟注/看牌，差则弃牌
  - 筹码不足时自动全下

- **人机状态同步**
  - 人机玩家状态正确广播给所有客户端
  - 人机标识: `isBot: true`，紫色边框 + 🤖徽章
  - 游戏结算时人机自动继续（筹码≥1000）

#### 技术实现
```javascript
// 后端: backend/games/poker/poker.socket.js
- createBotPlayer(seatNumber, roomId) - 生成人机玩家
- botAction(game, io, player) - 人机AI决策
- processGameDecisions() - 处理人机自动继续逻辑

// 前端: frontend/games/poker/room.html, game.html
- 人机头像显示（紫色边框）
- "添加人机"按钮（仅房主可见）
```
